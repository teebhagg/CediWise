# Contributing to CediWise

Thanks for helping improve CediWise.

## Before You Start

- Open an issue first for large features, bugs, or proposals.
- Keep changes focused and small when possible.
- Do not include secrets, API keys, or private credentials in any commit.

## Licensing

All contributions are covered by the [CediWise Contributor License Agreement](CLA.md).
By opening a pull request, you agree to the CLA.

## Workflow

1. Fork the repository or create a branch.
2. Make your changes.
3. Add tests or documentation when useful.
4. Open a pull request with a clear summary of what changed and why.

## What We Prefer

- bug fixes
- calculator and finance logic improvements
- documentation improvements
- Ghana-specific correctness fixes
- accessibility and performance improvements

## What We Cannot Accept

- secrets or credentials
- copied code that you do not have rights to contribute
- changes that violate the community or commercial licensing model

## Questions

If you want to contribute commercially or discuss relicensing, use the commercial licensing contact on the website.
